ApnaOS

A minimalistic OS in C performing memory, process, and file management to teach core operating-system concepts.

About ApnaOS

ApnaOS is a learning-oriented operating system written entirely in C and assembly. It demonstrates how fundamental OS components work together, with minimal dependencies and clear structure.

  • Bootloader & Kernel Setup: Loaded via GRUB, initializes GDT, switches to protected mode, and jumps into the kernel.
  • Kernel Architecture: Modular codebase with abstraction for memory, processes, and interrupt handling.
  • Process Management: Preemptive round-robin scheduler with PCB, context switching, and task isolation.
  • System Calls: Uses software interrupts to safely invoke kernel services from user space.
  • Interrupts: Custom IDT and IRQ remapping; ISRs and IRQ handlers for timer, keyboard, and faults.
  • Memory Management: Paging-based memory management with support for allocation, mapping, and CR3 switching.
  • File System: Minimal VFS layer supporting hierarchical directories and flat file operations.
  • CLI Interface: A basic shell to execute commands, invoke syscalls, and inspect system state.
  • Debugging & Diagnostics: Trap handlers, stack dumps, and serial output for development insights.

Each module is self-contained and simple enough for step-by-step walkthroughs, making ApnaOS ideal for OS design education and kernel hacking.
