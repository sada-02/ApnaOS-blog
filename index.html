ApnaOS

A minimalistic OS in C performing memory, process, and file management to teach core operating-system concepts.

About ApnaOS

ApnaOS is a learning-oriented operating system written entirely in C and x86 assembly. It demonstrates how fundamental OS components work together, with minimal dependencies and clear structure. Built from scratch with educational intent, ApnaOS provides hands-on experience with low-level system programming.

Core Features

  • Bootloader & Kernel Setup: GRUB-based Multiboot bootloader initializes GDT, IDT, switches to protected mode, and jumps into kernel space.
  • Kernel Architecture: Modular C codebase with clean abstractions for memory, processes, filesystem, and interrupt handling.
  • Process Management: Round-robin scheduler with Process Control Blocks (PCB), context switching, and support for fork/wait syscalls.
  • System Calls: Software interrupt-based syscall interface (fork, wait, yield, exit) for safe kernel service invocation.
  • Interrupt Handling: Custom IDT with ISRs, IRQ remapping via PIC, timer interrupts (PIT at 100Hz), and keyboard interrupt handling.
  • Memory Management: Paging-based virtual memory with page tables, heap allocation, and per-process memory isolation.
  • File System: Inode-based filesystem with hierarchical directories, file operations (create, read, write, delete), and user ownership.
  • Multi-User System: Password-protected user spaces with isolated file systems, authentication using password hashing, and root user support.
  • VGA Text Mode: 80x25 character display with scrollback buffer (1000 lines), navigation via Page Up/Down/ESC keys.
  • Command History: Per-user command history (20 commands) with up/down arrow navigation across sessions.
  • CLI Interface: Interactive shell supporting file/directory operations, process management, user management, and system commands.
  • Path Traversal: Full hierarchical path support with cd path/to/dir, cd .., and cd ~ commands.
  • Debugging & Diagnostics: Serial output for debugging, exception handlers, and comprehensive error reporting.

Recent Enhancements

  • Implemented scrollback buffer with Page Up/Down navigation for viewing output beyond screen size
  • Added multi-user authentication system with separate file spaces per user
  • Per-user command history context switching when changing users
  • Test processes for demonstrating fork/wait syscalls (simplesys, multisys)
  • Timer-based shutdown with 10-second countdown on exit command
  • Enhanced keyboard handling with extended scancode support
  • Help command with comprehensive documentation of all available commands

Each module is self-contained and documented with clear code examples, making ApnaOS ideal for learning OS internals, kernel development, and system programming concepts.
